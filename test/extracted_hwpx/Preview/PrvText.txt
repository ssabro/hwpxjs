개요
이 장에서는 본 연구에서 개발한 보안 점검 프로토타입의 개요에 관해 설명한다. 
보안 점검 프로토타입은 [그림 1]과 같이 일반적인 웹 서비스의 형태인 프론트 엔드 / 백 엔드 구조를 갖는다. 


[그림 2]는 프로토타입의 동작 흐름을 나타낸다. 사용자는 웹 브라우저를 통해 프론트 엔드의 웹 서버에 접근하여 네트워크 토폴로지 JSON 파일을 편리하게 생성하고 관리할 수 있으며, 생성한 네트워크 토폴로지를 시각적으로 확인할 수 있다. 사용자는 웹 브라우저를 통해 백 엔드의 API 서버로 어택 그래프 분석 모듈 구동과 관련된 정보를 전송한다. 백 엔드의 API 서버는 전달받은 네트워크 토폴로지 데이터와 기제작된 공격 시나리오 데이터를 기반으로 어택 그래프 분석 모듈을 구동한다. 어택 그래프 분석 모듈은 자산별 취약점 정보 매핑, 자산 취약성 / 위험도 점수 계산, 공격 경로 생성, 공격 경로 확률 계산, 공격 시나리오 타당성 점수 계산, 공격 그룹 도출 등의 기능을 수행한다. 백 엔드의 DB 서버는 어택 그래프 분석 모듈 구동에 필요한 취약점 데이터와 MITRE ATT&CK 데이터를 제공한다. 백 엔드의 API 서버는 사용자의 브라우저로 어택 그래프 분석 모듈 구동 결과를 전송하며, 사용자는 웹 브라우저를 통해 어택 그래프 모듈이 생성한 공격 경로 및 공격 시나리오에 관한 정보를 확인한다.
